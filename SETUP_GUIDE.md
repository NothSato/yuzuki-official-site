# ゆづき公式ブログ セットアップガイド

## 1. 開発サーバーの起動

### Next.js開発サーバー
```bash
npm run dev
```
- サイト: http://localhost:3000

### Sanity Studio
```bash
npx sanity dev
```
- CMS管理画面: http://localhost:3333

または、統合版Studio (http://localhost:3000/studio) を使用

## 2. 初期データの作成

### カテゴリーの作成
Sanity Studio で以下のカテゴリーを作成してください：

1. **iPhone**
   - タイトル: iPhone
   - スラッグ: iphone
   - 説明: iPhoneに関する情報、使い方、トラブル解決など

2. **Android**
   - タイトル: Android
   - スラッグ: android
   - 説明: Androidスマートフォンの機能解説、設定方法など

3. **アプリレビュー**
   - タイトル: アプリレビュー
   - スラッグ: app-review
   - 説明: おすすめアプリの紹介とレビュー

4. **スマホアクセサリー**
   - タイトル: スマホアクセサリー
   - スラッグ: accessories
   - 説明: スマホケース、充電器などのアクセサリー情報

5. **通信・格安SIM**
   - タイトル: 通信・格安SIM
   - スラッグ: mobile-plan
   - 説明: 格安SIM、通信プランの比較と解説

## 3. デプロイの準備

### 環境変数の設定
`.env.local` ファイルを作成し、以下を設定：

```
NEXT_PUBLIC_SANITY_PROJECT_ID=your_project_id
NEXT_PUBLIC_SANITY_DATASET=production
SANITY_API_TOKEN=your_api_token
```

### Vercelでのデプロイ
1. GitHubリポジトリを作成してコードをプッシュ
2. Vercelでプロジェクトをインポート
3. 環境変数を設定
4. デプロイ実行

## 4. サイト構成

- **トップページ** (`/`): ヒーローセクション + 最新記事3件
- **ブログ一覧** (`/blog`): 全記事の一覧表示
- **個別記事** (`/blog/[slug]`): 記事詳細 + 関連記事
- **プロフィール** (`/about`): ゆづきさんの自己紹介
- **Sanity Studio** (`/studio`): CMS管理画面

## トラブルシューティング

### よくある問題
1. **Sanityが接続できない**: プロジェクトIDとデータセット名を確認
2. **画像が表示されない**: Sanity CDNの設定を確認
3. **ビルドエラー**: 型定義とSanity クライアントの設定を確認
